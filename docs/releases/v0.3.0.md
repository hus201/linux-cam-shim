# cam-shim v0.3.0

**Research prototype — [linux consumer labs](https://github.com/linux-consumer-labs)**

Early-stage release: a userland compatibility layer for UVC webcams that break browsers and video calls on Linux. Detects cameras that advertise non-standard frame rates (e.g. 25 fps only), exposes a virtual **`{camera} - Shim`** device via v4l2loopback, and manages it on hotplug.

> Not a stability guarantee yet. Test on a non-critical system first. Requires **root** for `serve` / `fix` and the v4l2loopback kernel module.

## Highlights

- **`cam-shim serve`** — supervisor with netlink hotplug, settle retry, fallback poll, worker health, and circuit breaker
- **`cam-shim scan` / `status`** — physical vs virtual pairing and `recommended_devices` (JSON)
- **Stable loopback index** — same USB camera keeps `/dev/video10+` across replug and reboot (`/var/lib/cam-shim/devices.json`)
- **Direct capture → loopback relay** — MJPEG or uncompressed YUYV/NV12; loopback fps metadata set to `target_fps`
- **Recovery tools** — `doctor`, `clean`, `restore` for ghost nodes and orphan loopbacks
- **`.deb` package** — binary, systemd unit (disabled by default), docs

## Quick start

```bash
# .deb (Debian/Ubuntu amd64)
sudo dpkg -i cam-shim_0.3.0_amd64.deb
sudo cam-shim serve

# or standalone binary
chmod +x cam-shim
sudo ./cam-shim serve
```

Pick **`Your Camera - Shim`** in your app, not the raw physical device.

Ensure v4l2loopback is available (`modprobe v4l2loopback`). On kernel 7.x, prefer the in-tree module over `v4l2loopback-dkms`.

## Full changelog

### Added

- Netlink hotplug with 2s settle retry and configurable fallback poll
- Device registry for stable loopback assignments (`/var/lib/cam-shim/devices.json`)
- `scan`, `status`, `doctor`, `clean`, `restore` commands
- Camera view / pairing UX with `recommended_devices` in JSON output
- MJPEG and uncompressed YUYV/NV12 relay paths
- Frame pacing and improved loopback output handling
- Runtime state file (`/run/cam-shim/state.json`)
- Soak test script (`scripts/soak.sh`)
- `.deb` packaging with systemd unit

### Changed

- Virtual device label: `{Original Name} - Shim` (was `{name} - Linux Std` in earlier builds)
- Direct relay at native capture rate; loopback advertises `target_fps` metadata
- Loopback devices prefer `/dev/video10+` to leave low indices for physical cameras
